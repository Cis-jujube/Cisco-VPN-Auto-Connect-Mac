5. 能否扩大应用范围？

可以，但需要分层移植，不是简单复制 vpnc 或反过来。

  • 连接引擎（可共享思路）：两边本质都是驱动 Cisco Secure Client 的 -s 非交互模式。Mac/Linux 可移植 PowerShell 里的「分步 stdin + 隧道检测」逻辑到 Bash/Python，路径改为 /opt/cisco/secureclient/bin/vpn。
  • 存储层（必须换实现）：DPAPI 仅 Windows。跨平台方案：
    • macOS：security find-generic-password / Keychain
    • Linux：secret-tool（libsecret / GNOME Keyring）
    • 通用：用户主密码 + age / libsodium 加密 JSON（可跨机，需输入 passphrase）
  • GUI：Tkinter 在 Mac/Linux 可用，但需改路径探测、进程管理、图标逻辑。
  • 最小扩展路线：抽象 CredentialStore 接口（encrypt/decrypt/load/save）→ 保留现有 PowerShell 连接核心 → 新增 connect.sh + Keychain/secret-tool 后端。
  • 与 vpnc 的关系：vpnc 可作为 Duke Mac 用户的极简参考，但其明文存储不应照搬；本项目可在 Unix 侧提供「同等便捷 + 更强加密 + 多 profile」的升级版。
